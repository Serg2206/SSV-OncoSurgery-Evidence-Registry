# Security Policy

## Reporting a Vulnerability
Please report suspected vulnerabilities privately via GitHub Security Advisories for this repository.

## Sensitive Data Policy
- This repository must not contain PHI or direct identifiers.
- Any accidental sensitive data commit requires immediate rotation, removal, and incident logging.

## Hardening Baseline
- Least-privilege access for collaborators.
- Branch protection on `main`.
- CI checks required before merge.
- Dependency updates and periodic security review.

## Disclosure
After validation and mitigation, issues may be disclosed with remediation guidance.
