# Contributing

## Scope
We welcome contributions in:
- clinical registry schema quality,
- validation scripts,
- analytics and reproducibility,
- interoperability mappings (FHIR/OMOP),
- documentation and deployment.

## Basic Workflow
1. Open an issue describing the change.
2. Create a feature branch.
3. Add tests or validation artifacts where applicable.
4. Submit a pull request with clear clinical and technical rationale.

## Data Safety Requirements
- Never commit PHI or direct identifiers.
- Use de-identified sample data only.
- Explain limitations and assumptions for all analysis changes.

## Review Expectations
- Clinical review for protocol and outcome definition changes.
- Data governance review for schema and mapping changes.
- Technical review for pipeline and deployment changes.
